# Phase 2: shadcn/ui Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adopt shadcn/ui across student and teacher portals — init CSS variable theming, reconcile existing primitives, add Avatar/DropdownMenu/Toast/Charts/Skeleton/Dialog/Sheet, and install remaining components for Phase 3.

**Architecture:** Run `shadcn init` inside `frontend/`, map `--primary` to brand red `#eb1b23`, update existing Button/Card/Input to use CSS variables, then add new components via `npx shadcn@latest add`. No changes to the landing page or the backend. All new component files land in `frontend/src/components/ui/`.

**Tech Stack:** React 19, Vite, Tailwind CSS v3, shadcn/ui (default style), recharts (via shadcn chart), sonner, lucide-react (already installed).

```ts
export function getInitials(name: string): string {
  return name
    .split(' ')
    .map(n => n[0])
    .filter(Boolean)
    .slice(0, 2)
    .join('')
    .toUpperCase();
}
```
